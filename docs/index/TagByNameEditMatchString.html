Tag by Name Window

The "tag by name window" is used to write tags on selected files using the file name value. You can perform case conversions, and edit the "expected file name format" to face particular cases.

Read the "configuration examples" in the content window to fully understand how this window can be used in all its features.

Match String

The "match string" represents the string that is used with the file names to retrieve the tag values that have to be written.

For example if the field name is the following one:

File name:             "01 - Kate Bush - Running up that hill.mp3"

... the expected file name format must be the following string:

Expected format:    "< track > - < artist > - < title >"

You can do it by hand or you can select the correct fields, putting them in the correct order, and selecting the field separator " - ". Sometimes it not possible to do it without editing the match string. For example in this case: File name:             "01.Running up that hill.mp3"

.. you should select the correct fields and then select the field separator "." or " - ". Then edit the match string until it is the following one:

Expected format:    "< track >.< title>"
