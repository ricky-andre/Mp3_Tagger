Organizer Window

This panel lets you reorganize your files in directories and subdirectories with a name that is given by the tag values.

As usual, the best way to explain things is an example.

Reorganization example

Suppose that you have a lot of directories inside the following directory:

"c:\Documenti\Music\"

All the directories have the same name format, something like this:

"Bon Jovi - These Days"

"Green Day - Compilation"

"Mauro Picotto - Trance 2002"

"DJ Parade 2002 - Compilation"

"U2 - Achtung Baby"

"U2 - Rattle And Hum"

"U2 - The Joshua Tree"

... and many others.

Suppose now that you don't want to have music of a lot of different types in the same root directory (for example disco music in the same directory of rock music). Moreover, you want to change the directory names so that if you have a lot of albums of the same artist, the release year is included in the directory name, so that you can order the artist's albums from the oldest to the newest one. The directory structure should be something like this:

  • "genre"
    • "artist"
      • "artist - year - album"
These are the string matches you should select in the "Organizer window".

Reorganize window options

Select the fields you are interested to on the top of the window, and press the "add string" button to add the string listed in the text field. You can also edit the match string before adding it to the list, to insert fixed fields.

When you have something like the configuration of the example, on the left panel select the directories you want to reorganize.

On the bottom of the window, select if you want the files to be copied or moved, and select the starting directory where all the selected files have to be reorganized. As in most of the other panels, you can even choose the "recurse subdirectories" and and the "apply advanced search" option. Before choosing to apply the advanced search, read carefully the description of this feature inside the "Database creator window" help files.

Press the "try" button to have a preview of the way the selected files will be reorganized.

If you press the try button, no file will be moved nor copied anywhere. "Mp3 Studio" will read the tags from the selected directories, only from the files that have ".mp3" extension (not case sensitive), and will show with a directory tree where the files will be moved or copied.

In the above example, if all the files have correct tags, you will see that the main directory would be reorganized in the following way:

  • "Rock"
    • "Bon Jovi"
      • "Bon Jovi - 1998 - These Days"
    • "Green Day"
      • "Green Day - 2000 - Compilation"
    • "U2"
      • "U2 - 1992 - Achtung Baby"
      • "U2 - 1987 - Rattle And Hum"
      • "U2 - 1986 - The Joshua Tree"
  • "Disco"
    • "Mauro Picotto"
      • "Mauro Picotto - 2002 - Trance 2002"
    • "DJ Parade"
      • "DJ Parade - 2002 - Compilation"
You can even click the directories of the tree to see what files will be put inside it.

If you are satisfied of the results, you can press the "execute" button. The files will be moved/copied, and for every file that is processed, a line will be added to the response window: a green sphere means that everything has gone right for that file, while a red sphere means that an error has occurred for that file. A possible reason for a failure could be that the file has invalid characters (for example "?" or "/" on windows systems).

The old directories are not moved nor deleted, since inside them you could have ".jpg" images of the covers, and you could later on move them in the correct directory.
